Document Tags

Tag Name Description
<head> Head Metadata about website
<body> Body Body of Website
<header> Header Important info, top of page
<main> Main Primary content of page
<footer> Footer Info after content, bottom of page
<nav> Navigation Can go into header, footer, or be separate (use lists for content)
<title> Title (Metadata) Name of page, in <head>
<link> Link (Metadata) Links CSS document to HTML; requires
href="..." type="text/CSS" rel="stylesheet" attributes
<article> Article Independent units of content (individual blog post; comments)
<aside> Aside Secondary content
<section> Section Series of related content pieces
<menu> Menu Toolbar of buttons/tools (use <list> for content)

Attributes

Attribute Name Description
href="..." Hyperlink Reference Used in <a> tags
src="..." Source Used to identify the source of all media elements

Text

Tag Name Description
<h1> Header 1 Page title; most important info; only one per page
<h2> - <h6> Header 2 - Header 6 Content sub-headings; must be nested in decreasing order; can't skip numbers
<a> Anchor Hyperlink; requires href="..." attribute
<p> Paragraph Paragraph or general text
<cite> Citation Use for citing source of quotations, URL, book, movie title
<blockquote> Block quote Large, standalone quote, often in middle of article
<em> Emphasized Slightly more important text; screen readers will change voice; use CSS instead if only styling
<strong> Strong Much more important text; screen readers will change voice; use CSS instead if only styling
<address> Address Contact information (address, email, telephone)
<br> Break Use to create a meaningful line break (as in an <address>); use CSS instead if only styling

Image & Media Tags

Tag Name Description
<img> Image Photo or picture; requires src="..." and alt="..." attributes
<video> Video Video content; requires src="..." and alt="..." attributes
<audio> Audio Audio content; requires src="..." and alt="..." attributes
<embed> Embedded Any embedded media content; good for a gif; requires src="..." and alt="..." attributes
<figure> Figure Wrapper for any self-contained media; usually contains <figcaption> as child element
<figcaption> Figure Caption Must be child of <figure>

Using semantic tags helps to define the structure of the webpage, improve the browser's ability to interpret content, and increase accessibility.
